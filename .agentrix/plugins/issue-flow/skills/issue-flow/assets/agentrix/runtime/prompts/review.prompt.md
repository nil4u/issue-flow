目标：对当前 PR/MR 执行自动代码审查，并将审查结果发布到 PR/MR。

执行要求：
- 以 code review 立场优先检查 bug、行为回归、安全/数据风险、并发或边界条件问题、测试缺口。
- 审查 PR/MR diff、提交内容、相关代码和现有测试；不要只复述 PR/MR 描述。
- 发现问题时按严重程度排序，给出可定位的文件/行或代码符号引用。
- 如果没有发现明确问题，说明未发现阻塞问题，并列出仍存在的测试缺口或残余风险。
- 将结果发布为 PR/MR 评论；不要修改 source issue label、不要推进 `flow::`、不要更改 `automation::`。
- 除非审查本身需要验证，否则不要改代码；如果运行了验证命令，在评论中写明命令和结果。
